# 题目描述

先给出顺序统计量函数 SELECT(A,p,r,i) 的伪代码，它返回数组 A[p...r]A[p...r]A[p...r] 中第 iii 小的元素：

函数的第一行检查递归的基本情况，即 A[p...r]A[p...r]A[p...r] 中只包含一个元素，在这种情况下 iii 必然等于 111，直接返回 A[p]A[p]A[p]；第三行的 PARTITION(A,p,r) 自动将数组分为 A[p...q−1]A[p...q−1]A[p...q−1] 和 A[q+1...r]A[q+1...r]A[q+1...r] 两部分，并返回分界元素的下标 qqq，其中 A[p...q−1]A[p...q−1]A[p...q−1] 中的每个元素都小于或等于 A[q]A[q]A[q]，A[q+1...r]A[q+1...r]A[q+1...r] 中的每个元素都大于 A[q]A[q]A[q]；第四行计算子数组 A[p...q]A[p...q]A[p...q] 中的元素个数 kkk，如果 kkk 等于 iii，那就说明 A[q]A[q]A[q] 就是 A[p...r]A[p...r]A[p...r] 中第 iii 小的元素，直接返回；第七行中，如果 iii 小于 kkk，说明第 iii 小的元素在前一半区间中，数组 A[p...r]A[p...r]A[p...r] 中第 iii 小的元素就是数组 A[p...q−1]A[p...q−1]A[p...q−1] 中第 iii 小的元素；第九行中，否则，说明第 iii 小的元素在后一半区间中，数组 A[p...r]A[p...r]A[p...r] 中第 iii 小的元素就是数组 A[q+1...r]A[q+1...r]A[q+1...r] 中第 i−ki−ki−k 小的元素（因为保证了前一半数组中的元素和 A[q]A[q]A[q] 都比后一半数组中的元素小）。

第二行 nnn 个整数，第 iii 个正整数即为数组的第 iii 个元素 AiA_iAi​，保证 −107≤Ai≤107−10^7 \le A_i \le 10^7−107≤Ai​≤107，且所有元素互不重复。

# 输出格式

一个整数，表示原数组中第 kkk 小的元素。

# 题解：快速排序 + 模拟

这个题直接给出了这个算法的伪代码，所以写起来非常好写。其实这个算法就是 C++ 中的 nth_element() 函数的原型，该函数就是在线性的时间复杂度内求出数组的第 kkk 大 / 小数。关于时间复杂度的证明：

对于随机数据，期望每经过一次 PARTITION 操作后都能将数组对半分，每次 PARTITION 操作的复杂度为区间长度，因此 n+n2+n4+⋯→2nn + \dfrac n 2 + \dfrac n 4 + \cdots \to 2nn+2n​+4n​+⋯→2n，因此时间复杂度期望是线性的。
