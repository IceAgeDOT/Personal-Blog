# 问题引入

如果你遇到了一类这样的问题：

有 nnn 个物品，你必须在遵守一定限制的前提下从中选出 mmm 个物品，最大 / 小化这 mmm 个物品的总价值。

你可能会想到动态规划，但是这样动态规划时间复杂度是 O(nm)O(nm)O(nm) 起步的。

但如果这个问题去掉 mmm 个物品的要求，你可以直接用一维动态规划，或许会更容易解决；而且你发现，如果要求选 xxx 个物品的情况下最大 / 小价值为 g(x)g(x)g(x)，函数 g(x)g(x)g(x) 具有上凸 / 下凸的性质（实际上大多数这类问题都有这个性质），那么你可以用到一个利器来优化朴素的二维 dp\text {dp}dp：wqs\text {wqs}wqs 二分。

我们总能切到一个最高点，设这个点是 (x,g(x))(x, g(x))(x,g(x))，直线斜率为 ccc，那么有如下等式成立：

g(x)=cx+bg(x) = cx + b g(x)=cx+b

那么 −cx-cx−cx 有什么实际含义吗？我们可以理解为 把每个物品的价值都减去一个 ccc，则我们选出的 xxx 个物品总价值就正好减去了 cxcxcx！

如果我们一开始设定了这条直线的斜率为 ccc，然后我们把每个物品的价值都减去 ccc，进行一次不限定物品个数的一维 dp\text {dp}dp，能找到一个最大值，这个最大值就对应着这条相切直线的截距，如果我们在 dp\text {dp}dp 的过程中还记录下我们选了多少个物品，就能很容易地反推出 g(x)g(x)g(x) 是多少！

于是我们的优化思路就出来了：我们二分这条直线的斜率，找到一个斜率使直线能和我们需要的 (k,g(k))(k, g(k))(k,g(k)) 相切，我们就能反推出 g(x)g(x)g(x) 是多少！

关于斜率的上下界，我们找到 (0,g(0)),(1,g(1))(0, g(0)), (1, g(1))(0,g(0)),(1,g(1)) 之间的斜率和 (n−1,g(n−1)),(n,g(n))(n - 1, g(n - 1)), (n, g(n))(n−1,g(n−1)),(n,g(n)) 之间的斜率就可以确定下来。

大体思路就是这样，时间复杂度为 O(nlog⁡p)O(n \log p)O(nlogp)，其中 nnn 为物品个数（此处我们假定一维 dp\text {dp}dp 是线性可做的），ppp 是价值的值域。但我们还有一个细节问题需要说明。如果我们的图长这样：
